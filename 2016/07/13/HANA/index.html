Unit 1 Motivation & Basic Concepts

Basic Concepts

  1. 数据库管理系统

    DBS=DB+DBMS

    HANA is one

    Database-Management-System √

    Database ×

  2. 3 Level Schema Architecture

    External Level（最终看到和使用的局部数据的逻辑结构和特征的描述，是数据库用户的数据视图，是和某个应用有关的数据的逻辑表示） > Conceptual Level（相当于逻辑层，也叫逻辑模式，是对DB中全体数据的逻辑结构和特征的总体描述） > Internal Level（可以理解为磁盘上的数据，关乎数据的物理结构和存储方式）

  3. HANA数据库模型的特点

    • 可以基于行也可以基于列

    • 是一种关系型数据库

      R ⊆Ax B x C

      

    • SAP HANA is a relational data base management system and SAP HANA supports SQL

Unit 2 Reading data from a table or view

SELECT

  1. ‘’ 表示常量，读出来还是一样
  2. “” 表示包起来一个名字
  3. 不加任何引号，自动把小写转化成大写，所以推荐不加引号，这样避免区分大小写
  4. 如果有特殊字符，如空格，必须加双引号
  5. AS可以省略，写在属性后面即可重命名
